## LiDAR + IMU + Encoder 데이터 결합으로 위치 보정 

### ✨ 센서별 특징 및 역할
| 센서 | 데이터 | 역할 | 한계 |
| --- | --- | --- | --- |
| **LiDAR** | 주변 환경 3D/2D 포인트 클라우드 | **절대적인 위치** 제공 및 **SLAM** (Simultaneous Localization and Mapping) 기반의 정확한 위치 추정. | 노이즈, 환경 변화에 취약, 계산량 많음. |
| **IMU** | 가속도 및 각속도 | **단기적인 움직임** 추적, 특히 빠른 자세 변화 및 **관성 항법**에 유용. | **드리프트(Drift)** 현상 발생 (시간이 지남에 따라 오차가 누적됨). |
| **Encoder** | 바퀴 회전 수 | 로봇의 **상대적인 움직임** (오도메트리) 추정. | 바퀴 미끄러짐(**Slippage**) 발생 시 오차 발생. |


### 현재 프로젝트의 상황 
- 센서 퓨전 방식 적용은, 현재 저희가 보유중인 장비로는 lidar랑 depth camera, imu가 있는데 차량에 설치 되어 있지는 않아서, 부착한 후에 **depth carmera로 vision slam이나 obstacle detection**을 하는 방법이 가능할 것 같습니다. 
- 추가적으로 **odom 보정을 imu 기반**으로 할 수도 있을거 같습니다.
